# BiasRAG — Bias駆動型特化Latent Space設計

**法律ドメインで実装・検証済みのアーキテクチャを、自動運転安全判断へ転用する研究ポートフォリオ**

---

## このリポジトリについて

通常のRAGは「テキストの意味的な近さ」で検索する。

このプロジェクトは発想を逆転させ、**「評価軸（Bias）のスコアの近さ」で検索する**アーキテクチャを提唱・実装した。

LLMがドメイン知識を評価軸に変換し、そのスコアベクトル自体がLatent Spaceを形成する。  
このLatent Spaceは**評価軸を置き換えるだけで、任意のドメインに転用できる**。

---

## アーキテクチャ概要

```
入力文（自然文）
  ↓ LLM：ドメイン固有の評価軸でスコアリング
5軸スコアベクトル（抽象的な「争いの構造」）
  ↓ ChromaDB：スコア距離で類似ケース検索
類似JSONを取得（具体的事実・証拠）
  ↓ LLM：γパラメータで参照度を制御して生成
出力（有利/不利の判断・根拠付き）
```

### γ（ガンマ）パラメータ

| γ値 | 動作 |
|-----|------|
| 0.0〜0.3 | 検索ケースの具体的事実をそのまま引用 |
| 0.4〜0.6 | 入力内容に合わせて読み替え |
| 0.7〜1.0 | 争点の構造だけを抽象的に翻訳（類推モード） |

## Phase 1（実装済み）：BiasRAG-Law

**法律ドメインで動作確認済み。**

### 5軸評価（法律版）

| 軸 | 意味 |
|----|------|
| `shoko_sonzai` | 主張を裏付ける証拠・書面・記録の有無 |
| `aite_hitei` | 相手方が事実を否定・争っている度合い |
| `jijitsu_meikaku` | いつ・誰が・何をしたかの明確さ |
| `daisan_kaizai` | 代理人・仲介者等の第三者の介在 |
| `horitsu_konkyo` | 条文・契約書・判例等の法的根拠の強さ |

### データ構造（JSONの3層）

```json
{
  "形式情報": "case_id / case_name / tachiba / subject",
  "検索スコア": "shoko_sonzai / aite_hitei / jijitsu_meikaku / ...",
  "LLMへ渡す": "yuri_facts / furi_facts / tsukkomi / statute"
}
```

### 技術スタック

- **ベクトルDB**: ChromaDB（スコアを5次元ベクトルとして格納）
- **LLM**: Groq API（llama-3.3-70b-versatile）
- **実行環境**: Google Colab
- **データ**: 司法試験・予備試験問題ベースのフィクションJSONを手作成

## Phase 2（設計中）：BiasRAG-Safety → 自動運転への転用

**評価軸を「法律」から「安全性」に置き換えるだけで、同一アーキテクチャが自動運転判断に転用できる。**

### 5軸評価（自動運転安全版・設計中）

| 軸（法律版） | → | 軸（安全版） | 意味 |
|---|---|---|---|
| `shoko_sonzai` | → | `sensor_confidence` | センサー情報の信頼度・冗長性 |
| `aite_hitei` | → | `object_uncertainty` | 検出対象の挙動不確実性 |
| `jijitsu_meikaku` | → | `scene_clarity` | 交通シナリオの状況明確さ |
| `daisan_kaizai` | → | `external_factor` | 歩行者・他車・天候等の外部要因 |
| `horitsu_konkyo` | → | `rule_basis` | 交通法規・安全基準への準拠度 |

### 設計思想

```
交通シナリオ入力（センサーログ + 状況記述）
  ↓ LLM：Safety Biasで5軸スコアリング
安全性スコアベクトル
  ↓ ChromaDB：類似シナリオ検索
過去の類似ヒヤリハット・事故事例を取得
  ↓ LLM：判断根拠を説明可能な形で出力
「なぜその判断をしたか」が人間に説明できる安全判断
```

### このアーキテクチャの優位性

1. **説明可能性** — 判断根拠が5軸スコアと類似事例として出力される
2. **軽量** — LLMはスコアリングと生成のみ。判断の主体はBias-RAG
3. **エッジ転用可能** — 車載小型LLM（SWALLOWなど）への置換でオフライン動作
4. **ドメイン追加が容易** — 評価軸の再定義だけで新しい判断領域に拡張できる

## 実行方法

1. Google Colabで `BiasRAG_Law_v3.ipynb` を開く
2. Colabの「シークレット」に `GROQ_API_KEY` を登録
3. `sample_data/` をGoogle Driveのマイドライブ直下にコピー
4. 全セル実行
